# LivingCityEngine AI Studio

## Purpose

LivingCityEngine is developed using an AI-assisted studio workflow.

Multiple AI tools may contribute to the project, but all contributors must follow the same vision, architecture, and quality standards.

---

# Team Structure

## Creative Director / Product Owner

Human Founder

Responsibilities:

- Final creative decisions
- Feature priorities
- Gameplay direction
- Player feedback
- Release decisions

---

# Technical Director

ChatGPT

Responsibilities:

- Overall architecture
- System design
- Technical planning
- Gameplay design
- Code review
- Performance strategy
- Documentation
- Maintaining project consistency

---

# Implementation Engineer

Claude Code / Coding Agents

Responsibilities:

- Repository changes
- Feature implementation
- Refactoring
- Debugging
- Testing
- Code improvements

Rules:

Before changing code:

1. Read AGENTS.md
2. Read ARCHITECTURE.md
3. Review existing systems
4. Understand dependencies

# Development Workflow

## Planning

Before implementing a feature:

- Define purpose
- Define user experience
- Define technical requirements
- Identify affected systems

---

## Implementation

The implementation agent should:

- Follow existing architecture
- Create reusable systems
- Document changes
- Test functionality

---

## Review

Every major feature should be reviewed for:

- Gameplay quality
- Performance
- Maintainability
- Mobile compatibility

---

# Communication Rules

AI contributors should:

- Avoid conflicting designs
- Ask for clarification when requirements are unclear
- Prefer scalable solutions
- Keep documentation updated

---

# Project Philosophy

Build like a professional game studio.

Prioritize:

- Quality

- Player enjoyment
- Long-term maintainability
